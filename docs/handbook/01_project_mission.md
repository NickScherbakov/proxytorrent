# 1. Миссия проекта

## Цель главы

Понять, зачем существует ProxyTorrent, какую проблему он решает, и каковы основные принципы и ценности проекта.

## Миссия

ProxyTorrent предоставляет безопасный, изолированный и приватный способ получения веб-контента с последующим распространением через BitTorrent.

### Проблема

В современном интернете существует несколько вызовов:

1. **Приватность**: прямое обращение к веб-ресурсам раскрывает IP-адрес запрашивающего
2. **Цензура**: доступ к некоторым ресурсам может быть ограничен географически
3. **Безопасность**: прямая загрузка контента может быть небезопасной
4. **Эффективность**: многократная загрузка одинакового контента неоптимальна

### Решение

ProxyTorrent решает эти проблемы путём:

1. **Изоляция через прокси/VPN**: все запросы идут через настраиваемый прокси или VPN
2. **Упаковка в торренты**: контент упаковывается в приватные торренты
3. **Дедупликация**: content-addressable storage предотвращает дублирование
4. **Безопасность**: аутентификация, валидация контента, rate limiting

## Основные ценности

### 1. Безопасность превыше всего
- Все соединения через прокси/VPN (когда включено)
- HMAC-аутентификация запросов
- Валидация контента (MIME types, размер)
- Приватные торренты по умолчанию

### 2. Приватность пользователей
- Изоляция сетевого уровня
- Минимизация логирования чувствительных данных
- Опциональное шифрование торрентов

### 3. Простота использования
- REST API с понятными эндпоинтами
- Docker-контейнеризация для лёгкого развёртывания
- Подробная документация и примеры

### 4. Производительность и эффективность
- Асинхронная обработка запросов
- Content-addressable storage для дедупликации
- Настраиваемые ограничения и лимиты

### 5. Прозрачность и открытость
- Open source (MIT лицензия)
- Полная документация архитектуры
- Примеры использования

## Принципы проектирования

### Модульность
Система состоит из независимых компонентов (Fetcher, Packager, Seeder), которые можно тестировать и модифицировать отдельно.

### Настраиваемость
Все критичные параметры управляются через переменные окружения без изменения кода.

### Безопасность по умолчанию
- Аутентификация включена по умолчанию (в production)
- Приватные торренты
- SSL-верификация
- Rate limiting

### Fail-safe поведение
- Graceful shutdown всех компонентов
- Обработка ошибок без прерывания сервиса
- Retry-логика для временных сбоев

- Организации с требованиями к приватности и изоляции сети

## Use Cases

### 1. Безопасная загрузка контента
Клиент хочет загрузить контент с веб-сайта, не раскрывая свой IP-адрес.

**Решение**: ProxyTorrent загружает контент через прокси/VPN и возвращает торрент.

### 2. Распространение контента через торренты
Организация хочет получить веб-контент и распространить его внутренним пользователям через BitTorrent.

**Решение**: ProxyTorrent создаёт приватный торрент, который можно раздавать внутри сети.

### 3. Архивирование веб-контента
Нужно периодически сохранять снимки веб-страниц с сохранением приватности.

**Решение**: ProxyTorrent получает контент изолированно и сохраняет в content-addressable storage.

### 4. Обход географических ограничений
Доступ к контенту через прокси в другой стране.

**Решение**: Настройка ProxyTorrent с прокси-сервером в нужной юрисдикции.

## Нецелевое использование

ProxyTorrent **не предназначен** для:

- ❌ Массового скрапинга веб-сайтов (есть rate limiting)
- ❌ Нарушения авторских прав или условий использования сайтов
- ❌ DDoS-атак или других злонамеренных действий
- ❌ Обработки данных более 50 МБ без настройки лимитов
